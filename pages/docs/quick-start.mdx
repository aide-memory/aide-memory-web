# Quick Start

Get AIDE Memory running in your project in two minutes.

<Steps>

### Install

```bash
npx aide-memory init
```

This command:
- Creates the `.aide/` directory
- Installs six hooks (SessionStart, PreToolUse, PostToolUse, UserPromptSubmit, Stop, PreCompact)
- Writes rules files for Claude Code and Cursor
- Configures the MCP server
- Zero config required

### (Optional) Scan your project

Generate initial memories from your codebase structure:

```bash
npx aide-memory init --scan
```

This scans your project and creates initial memories for common patterns, dependencies, and architecture patterns.

### Work normally

Start working with your agent. Hooks capture corrections automatically:
- When you say "No, use composition here," the hook stores it as a memory
- When the context window fills, the PreCompact hook saves planning decisions
- When a task finishes, the Stop hook extracts final decisions

### Check what's been captured

```bash
npx aide-memory list
```

View all captured memories:

```bash
npx aide-memory list --scope src/components/**
npx aide-memory list --layer preferences
```

### Next session

Your agent gets nudges automatically:

```
3 memories exist for src/components/**. Call aide_recall if relevant.
```

That is ~20 tokens. The agent decides whether those memories matter for the current task.

</Steps>

## Common Commands

### List memories

```bash
# All memories
npx aide-memory list

# For a specific path
npx aide-memory list --scope src/checkout/**

# For a specific layer
npx aide-memory list --layer preferences

# Both
npx aide-memory list --scope src/** --layer technical
```

### Search memories

```bash
# Full-text search
npx aide-memory search "composition pattern"

# With path scope
npx aide-memory search "test utilities" --scope tests/**
```

### Delete a memory

```bash
npx aide-memory forget <memory-id>
```

### Reset everything

To reset memories and config, delete `.aide/` and re-run init:

```bash
rm -rf .aide/
npx aide-memory init
```

## Configuration

Configure aide-memory via:

```bash
npx aide-memory config <key>           # read
npx aide-memory config <key> <value>   # write
```

### Available settings

Common knobs:

- `embeddings.backend`, `"auto"` (default), `"transformers"`, `"ollama"`, or `"none"`
- `embeddings.model`, model to use (Transformers.js or Ollama, e.g. `"nomic-embed-text"`)
- `hooks.visible`, show/hide `aide-memory · ...` systemMessage lines in your terminal (default `true`). Flipping to `false` hides the surface without disabling the hooks underneath.
- `hooks.read.maxBlocks` / `hooks.edit.maxBlocks`, how many times per session pre-read / pre-edit hooks can hard-block (`0` to disable, `1` default)
- `hooks.search.mode`, `"soft"` (default), `"block"`, or `"off"` for the Grep nudge
- `hooks.correction.enabled`, `true`/`false` for UserPromptSubmit correction detection
- `hooks.precompact.mode`, `"cleanup"` (default) or `"off"`
- `memories.softening.threshold`, below this total-memory count, hard blocks become soft nudges (default `10`)
- `memories.defaultShared`, default `shared` value for new `preferences` memories. Per-call `shared: true|false` always overrides.

See the full config schema in `scripts/hooks/defaults.json` in the aide-memory repo.

## Using Multiple Editors

AIDE Memory works with Claude Code and Cursor. Install in any project:

```bash
npx aide-memory init
```

The same `.aide/` directory is read by both tools. Memories captured in Claude Code are available in Cursor immediately.

## Team Sharing

Commit memories to git:

```bash
git add .aide/memories/
git commit -m "Store team patterns and preferences"
git push
```

Team members get the memories via:

```bash
git pull
```

No Docker, no sync service, no authentication. Just git.

## Troubleshooting

### Memories not being captured

Check that the hook config files exist:

```bash
cat .claude/settings.json   # Claude Code
cat .cursor/hooks.json      # Cursor
```

If hooks are missing or stale, regenerate:

```bash
npx aide-memory init --update-rules
```

### Memory not showing up in next session

Verify the memory exists:

```bash
npx aide-memory list
```

Verify the path scope matches your current file:

```bash
npx aide-memory list --scope src/components/**
```

### Full-text search not working

Falls back automatically to LIKE-based search. To rebuild the SQLite cache (including FTS5) from JSON memory files:

```bash
npx aide-memory sync import
```
