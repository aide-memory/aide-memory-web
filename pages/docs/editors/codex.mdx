# aide-memory in Codex

**Status: rule template only at launch.** aide-memory's Codex adapter ships a curated rule template (same canonical body as Claude Code and Cursor, plus Codex-specific frontmatter) but does **not** yet generate hook or MCP config files at `aide-memory init`.

## What works today

- If you add aide-memory as an MCP server manually in Codex's MCP config, the seven MCP tools (`aide_recall`, `aide_remember`, `aide_update`, `aide_forget`, `aide_search`, `aide_memories`, `aide_import`) work identically to Claude Code.
- The rule template is generated at init.
- Nothing else automatic. No hooks, no pre-read blocks, no correction detection, no session-start injection.

## What's coming

Hook + MCP config generation at init is on the post-launch roadmap. Until the adapter ships, treat Codex as rules-template-only for planning purposes.

See [Supported Editors](/docs/supported-editors) for the honest accounting.
