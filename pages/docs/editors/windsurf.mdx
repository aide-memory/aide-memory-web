# aide-memory in Windsurf

**Status: rule template only at launch.** aide-memory's Windsurf adapter ships a curated rule template (same canonical body as Claude Code and Cursor, plus Windsurf-specific frontmatter) but does **not** yet generate hook or MCP config files at `aide-memory init`.

## What works today

- If you add aide-memory as an MCP server manually in Windsurf's MCP config, the seven MCP tools (`aide_recall`, `aide_remember`, `aide_update`, `aide_forget`, `aide_search`, `aide_memories`, `aide_import`) work identically to Claude Code.
- The rule template is generated at init.
- Nothing else automatic. No hooks, no pre-read blocks, no correction detection, no session-start injection.

## What's coming

Full hook + MCP config generation at init is in flight. Each editor adapter is a small file (template + config-shape), so adding Windsurf's full integration is bounded work; we just want to validate against Claude Code and Cursor first before generating Windsurf configs that might conflict with whatever you have set up.

If you'd like to track the work or pitch in, see [Supported Editors](/docs/supported-editors) for the capability matrix and the editor-onboarding playbook.
