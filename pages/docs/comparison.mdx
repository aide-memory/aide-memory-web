# Comparison

AIDE Memory, [claude-mem](https://github.com/thedotmack/claude-mem), and [engram](https://github.com/ayvazyan10/engram) all solve the same core problem — AI agents forget between sessions — but they take meaningfully different approaches. This page describes each tool factually so you can pick the one that matches your workflow.

## Feature Comparison

| Feature | AIDE Memory | claude-mem | engram |
|---------|-------------|------------|--------|
| **Capture mechanism** | Hooks pre-wired across 6 lifecycle events at `aide-memory init` | Session-end summarization that emits the next session's context | Opt-in session-end hook script you copy in and configure |
| **Recall mechanism** | ~20-token nudge per file read; agent decides which memories to fetch | Inject context from recent sessions at session start; 3-layer MCP search (search → timeline → get_observations) on demand | Namespace-scoped retrieval against typed memory store |
| **Scoping model** | Glob patterns with inheritance (e.g. `src/auth/**`) inside a project | Folder granularity via auto-generated per-project-folder Folder Context Files | Namespace per project or agent, with opt-in cross-namespace recall |
| **Memory structure** | 4 layers: preferences, technical, area_context, guidelines | Folder activity timelines + observations | 3 types: Episodic, Semantic (knowledge graph), Procedural |
| **Storage** | Local JSON files + SQLite cache | Chroma (vector DB) + Bun-managed worker service | Local store (per-namespace) |
| **Install** | `npx aide-memory@latest init` | `npx claude-mem install` (manages Bun + uv) | npm package |
| **Git sync** | Native — memories are files in `.aide/`, commit to share with the team | Not the primary distribution model | Not the primary distribution model |
| **Compaction handling** | PreCompact hook saves planning decisions before context is lost | Session-end summarization | Session-end hook (opt-in) |
| **License** | Proprietary freeware (free for solo developers) | AGPL-3.0 | MIT |
| **Price** | Free for solo developers (potential team/pro tiers later) | Free | Free |

## How AIDE Memory's Approach Differs

**Per-file nudge instead of session-start dump.** AIDE Memory injects a small (~20 token) nudge when the agent opens a file that has scoped memories, and lets the agent decide whether to call `aide_recall`. Other tools take different approaches — claude-mem injects context from recent sessions at session start and exposes search tools for on-demand detail; engram retrieves from typed namespaced stores. Different tradeoffs: per-file recall is precise but depends on file reads; session-start injection is comprehensive but pays a token cost up front.

**Glob-pattern path scoping inside a project.** A memory about test utilities can be scoped to `tests/**` so it surfaces only when the agent is in test code. claude-mem scopes by folder via Folder Context Files (per-folder CLAUDE.md activity timelines). engram scopes by namespace (per project or agent). Same goal — surface relevant context, suppress irrelevant context — at different granularities.

**Hooks pre-wired at init.** AIDE Memory configures hooks at six lifecycle events (SessionStart, PreToolUse, PostToolUse, UserPromptSubmit, Stop, PreCompact) the moment you run `aide-memory init`. engram offers an opt-in session-end hook script. claude-mem handles capture through its session lifecycle. If you want capture wired up without manual config, that's the AIDE Memory default.

**Files as the storage substrate.** Memories live as JSON files inside `.aide/` next to your code, so `git add` / `git commit` / `git pull` is the team-sync path. claude-mem and engram use vector / typed stores instead, which trade the file-level workflow for richer query semantics. Pick the model that matches how your team already works.

**Four typed layers.** preferences (how a contributor likes to work), technical (facts about the stack), area_context (decisions for a code area), guidelines (team principles). engram uses a different axis — Episodic / Semantic / Procedural — which is closer to a cognitive-architecture framing. Both are valid; the right one depends on the kinds of memories you want to capture.
